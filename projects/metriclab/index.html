The project

Finding the problem is only the beginning.

MetricLab follows the full product data science workflow: diagnose conversion behavior, propose an intervention, plan and validate an experiment, quantify uncertainty, monitor guardrails, and make a launch decision.

Observed analysis

Using event-level BigQuery SQL, I constructed an ordered session funnel and found the largest loss between product views and cart additions. Device and acquisition-source comparisons showed similar desktop and mobile performance, avoiding an unsupported mobile-friction claim. The evidence instead motivated testing clearer product information and a stronger add-to-cart call to action.

Experiment design

A two-proportion power calculator estimated that detecting an increase from 18.3% to 20.0% requires 8,409 sessions per variant. The framework includes sample-ratio-mismatch detection, confidence intervals, p-values, and revenue per session as a downstream guardrail.

Recommendation: Iterate. Cart conversion improved significantly, but the revenue confidence interval could not rule out meaningful harm.

Methodology boundary

The funnel and segmentation findings use real public GA4 data. The experiment outcome is simulated with a documented injected treatment effect and is not presented as evidence of a real product change.

  • Ordered session-level funnel built with BigQuery SQL
  • Statistical power planning and assignment-balance validation
  • Lift, uncertainty, significance, and revenue guardrail reporting
  • Interactive Streamlit dashboard with GitHub Actions CI
