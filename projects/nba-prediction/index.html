The project

From historical stats to an interactive prediction experience.

Player projections are most useful when they go beyond a spreadsheet. This project connects model development, a prediction API, and a custom interface into one complete product.

9player metrics forecasted
3connected product layers
1interactive web app

Approach

I trained a prediction model in Python with scikit-learn and NumPy, using historical performance and engineered inputs such as age trends and projected minutes. A Flask API makes the predictions available to the application, while a React frontend visualizes results and ranks players by predicted statistics.

What I built

  • A modeling pipeline for nine player performance metrics
  • Feature engineering for age trends, playing time, and historical performance
  • A Flask API for serving model predictions
  • A responsive React interface for comparison and fantasy ranking
