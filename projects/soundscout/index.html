The project

Music discovery shaped around the listener.

SoundScout turns a small set of songs into a normalized taste profile, ranks similar tracks, and explains why each recommendation fits—while keeping the playlist varied and responsive to the listener’s mood.

Recommendation system

The system combines danceability, energy, valence, acousticness, instrumentalness, tempo, genre similarity, and a small popularity signal. Users can seed the profile with up to five songs and steer it toward balanced, energetic, chill, feel-good, or acoustic mixes. Artist-diversity reranking prevents repetitive results, while dismissed tracks are excluded from later recommendations.

Product experience

I redesigned the Streamlit interface as a responsive music product with searchable song and artist selection, personalized taste descriptions, album artwork from the iTunes Search API, direct Spotify links, session-based likes and dismissals, adjustable playlist length, and custom CSV uploads.

Data and engineering

A reproducible preparation script converts the original 114,000-row source into an optimized 40,000-track catalog, removes duplicate Spotify IDs, and prioritizes popular tracks. The packaged Python project is continuously tested with GitHub Actions; the first automated run passed successfully.

Responsible data use: The catalog is derived from a CC0-designated Kaggle Spotify tracks dataset. It contains track metadata and audio features with real Spotify IDs, but redistributes no audio files. The project includes attribution, licensing notes, and reproducible preparation documentation.

  • Multi-song taste profiles with normalized audio features
  • Mood steering, artist diversity, dismissals, and explainable match scores
  • Offline evaluation for relevance, diversity, and catalog coverage
  • Responsive Streamlit application with tests, CI, and public deployment
